React基础

  • react 用于构建用户界面的渐进式JAVASCRIPT库，第一个真正意义上组件化思想
    • 本身只处理UI（数据驱动视图）
    • 不处理AJAX
    • 不关系路由

      react特点

  • 组件化
  • 高效，虚拟DOM
  • 灵活，渐进式，本身只处理UI，可以结合其他技术
  • 声明式设计
    • data响应式数据
    • methods处理函数
    • 特点组件生命钩子
    • 方法就是类成员
  • JSX
    • 一种预编译javascript语言，允许你的js和HTML混搭
    • 模板中就是JS逻辑
  • 单向数据流
    • 所有数据都是单向的，组件床底的数据都是单向的只能父组件=>子组件
    • vue也是单向数据流
    • 没有双向数据绑定

      react与vue对比

      • vues生产力更高，react技术占比比较重
      • 两个框架都采用虚拟DOM
      • 组件化
      • 数据绑定
    • 都支持数据驱动视图
    • vue支持表单控件的双向数据绑定
    • react不支持双向数据绑定
      • 核心库很小，渐进式js库
      • React采用JSX语法编写组件
      • Vue采用单文件组件
    • template，script，style

      react基础

      • 云端编程环境 只适用于demo测试
      • 脚手架工具 create-react-app
    • 类似于vue-cli
    • 集成了webpack
      • @babel/standalone 的作用是可以在浏览器中完成es6到es5的转换
    • 是一个独立的编译工具，支持调用它的api完成es6=>es5
      • react的核心理念：组件化，不支持管理dom模板，他必须把所有的模板放在组件中

        JSX环境 type=”text/babel”

      • 非模块化环境
      • babel-standalone
      • 执行编译，速度慢
      • 只是用开发测试环境
      • 模块化环境
      • 结合webpack配置babel响应的工具完成预编译
      • 浏览器执行的是预编译结果
      • babel-preset-react
      • jsx语法基本使用
      • 必须有一个根节点
      • jsx注释
    • 单行注释 {/**/}
    • 多行注释 { /*
      */}
      • 动态属性
      • class使用className对应类名
      • for使用htmlfor
      • 属性使用表达式 title={user.name}
      • 单标签要/闭合
      • 事件绑定 onClick={fname}
    • 使用驼峰命名
    • 必须绑定一个函数
    • 不能使用字符串方式，一定要使用{函数}来绑定

      React修改视图数据

      • this.setState({}) 不能使用this.state.propName
      • 命令行 hs -c-l -o

vue项目遇到的问题

在 build 项目时,报错 .plugins[0] may only be a two-tuple or three-tuple 检查配置文件babel.config.js发现 自己这样写

				plugins: [
			  [
			    'component',
			    {
			      libraryName: 'element-ui',
			      styleLibraryName: 'theme-chalk'
			    },
				 ...prodPlugins,
				'@babel/plugin-syntax-dynamic-import'
			  ]
			]
应该这样配置
		    plugins: [
			  [
			    'component',
			    {
			      libraryName: 'element-ui',
			      styleLibraryName: 'theme-chalk'
			    }
			  ],
				 ...prodPlugins,
				'@babel/plugin-syntax-dynamic-import'
			]
